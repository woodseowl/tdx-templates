screenbutton

This character style indicates a thing to be clicked: a button, a menu heading or menu item, a tab, or a choice in a list.

Apply this style to text that appears on an object on the screen (like Next or OK), not to the action you take (like select or click).

Do not include the ellipsis when naming a button or menu item. Say Click Add, not Click Add...

If the thing that gets clicked has no text (for example, an arrow), use an image of the thing inline or near the mention of the thing. Apply the screenbutton style to any text reference to the thing (for example, telling the user to click on the arrow to continue): “Click the large red arrow.”

Do not apply this style to descriptions as in this phrase: “highlight the person’s name.”

screentitle

This character style indicates a window title, dialog box name, column heading, label in the dialog box or window, or any words that serve as identifiers or markers. For example, “In the Options dialog box, in the Color section, select Red.” Note that the words “dialog box” (or “window” or “column heading”) do not have the style applied.

Do not apply this style to actions (like right-click or drag).

Some items are clickable but not clicked in the current setting. Use the style that fits the context. For example, "In the Last Name column, highlight the person’s name," but “click Last Name to sort the table by that column.”

typeditem

This character style indicates words (or numbers) that the user will type in. For example, “For Port enter 1058.”

Text appearing in this style should appear exactly as we expect the user to type it. Do not put the styled text in quotes. The style itself indicates the beginning and end of the text to be entered. Where feasible (and especially when the text to be entered is more than a few characters), place the text to be entered on a separate line, all by itself. For example,

  1. For User ID, enter
    cornell\netid@cornell.edu\

Do not use this style when referring to the field name. For example, do not use this style anywhere in this sentence: "Type your NetID and password into the boxes."

However, if the typed item includes a placeholder (for example, netid@cornell.edu), use the typeditem style, and emphasize the placeholder with bold.

Wherever possible, do not break this text across lines or pages.

Paths and filenames use this style, even if they’re entered through browsing rather than actual typing.

warning-box

Warning: warning

Used to convey information that will steer our users away from a mistake. The fact that we use the style makes the information important. Do not use "Important" or "Note" or "Warning" or anything else as a header or introduction to the text in the box.

tip

Note: info

Used to make a recommendation, especially where we've given people choices or left a decision up to them. The fact that we use the style makes the information a tip. Do not use "Important" or "Note" or "Tip" or anything else as a header or introduction to the text in the box.
